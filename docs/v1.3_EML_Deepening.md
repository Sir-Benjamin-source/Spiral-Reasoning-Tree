**Abstract**  
This v1.3 update deepens the integration of the eml operator within the Spiral Reasoning Tree. Building on the uniform binary gate introduced in v1.2, we now compute the full R_polish resonance score using nested eml-subtrees for multiplication, square-root, and overall composition.  

A new method, `get_resonance_provenance()`, provides explicit cross-referencing and bounded traceability, allowing the system to describe how the resonance value was derived (including method, seeds used, and estimated tree depth). This enhancement improves auditability and self-reference while remaining fully optional and backward-compatible with previous versions.

**Changes in v1.3**

1. **Deeper eml-tree usage in resonance computation**  
   The `compute_resonance()` method now constructs the entire R_polish formula through uniform eml gates where the eml gate is enabled. This moves the resonance engine closer to a single, coherent binary-tree grammar without sacrificing performance or flexibility.

2. **Provenance reporting**  
   Added `get_resonance_provenance()` which returns a structured dictionary describing the computation path. Example output when eml gate is active:  
   ```python
   {
       "method": "eml_gate_v1.3",
       "description": "Full R_polish computed via uniform eml binary trees (mul, sqrt, nested composition)",
       "seeds_used": {"positive": 1.0, "negative": -1.0},
       "tree_depth_estimate": "7–12 (depending on mul/sqrt expansion)",
       "resonance_value": 0.665...
   }
   ```  
   This feature directly supports bounded self-reference and makes resonance scores more interpretable for AI reasoning agents.

**Methodology & Compatibility**  
All changes were tested for symbolic coherency, recursive stability, and integration fit with the existing helical graph and SRM phases. The eml gate remains optional (`use_eml_gate=False` by default), preserving full backward compatibility with v1.1 and v1.2. No breaking changes were introduced.

By deepening the eml integration and adding explicit provenance, SRT v1.3 strengthens the hybrid advantage of the framework: the elegance of a single universal gate combined with the bounded, reciprocal, helical architecture of the Spiral Reasoning Model.
